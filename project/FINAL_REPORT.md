# 딥러닝 기반 반려동물 안구 질환 의심 예측 시스템 최종 보고서

## 1. 프로젝트 개요

본 프로젝트는 반려동물 눈 이미지를 입력받아 안구 질환 의심 가능성을 예측하고, 보호자에게 참고 가능한 질환 후보와 관리 정보를 제공하는 딥러닝 기반 이미지 분석 시스템이다. 사용자는 웹 화면에서 눈 이미지를 업로드하고, 모델은 이미지 품질을 점검한 뒤 정상 또는 질환 의심 여부와 상위 3개 질환 후보를 출력한다.

본 시스템은 수의학적 진단을 대체하지 않는다. 목적은 보호자가 이상 징후를 더 빨리 인식하고 병원 상담 여부를 판단하는 데 도움을 주는 보조 도구를 구현하는 것이다.

## 2. 필요성과 기술 가치

반려동물은 눈 통증이나 시야 이상을 직접 설명할 수 없기 때문에 보호자가 안구 질환을 늦게 발견하는 경우가 많다. 결막염, 백내장, 각막질환처럼 육안으로 비슷해 보이는 질환은 보호자 판단만으로 구분하기 어렵고, 치료 시기를 놓치면 증상이 악화될 수 있다.

이미지 기반 딥러닝 모델은 보호자가 촬영한 눈 이미지를 바탕으로 질환 후보를 빠르게 제시할 수 있다. 특히 본 프로젝트는 단순히 “질병 있음/없음”만 출력하지 않고, 의심 질환 후보, 질환 설명, 예방법을 함께 제공해 실제 사용자가 이해할 수 있는 형태로 결과를 구성했다.

## 3. 데이터셋

사용 데이터는 AIHub의 `반려동물 안구 질환 데이터`이다. 해당 데이터셋은 반려견과 반려묘의 안구 질환 이미지, 질환명, 질환 정도, 바운딩박스 및 세그멘테이션 라벨을 포함한다.

- 데이터 출처: AIHub
- 데이터명: 반려동물 안구 질환 데이터
- URL: https://aihub.or.kr/aihubdata/data/view.do?dataSetSn=562
- 데이터 형식: jpg 이미지, json 라벨
- 실제 사용 범위: Validation 원천데이터 중심

전체 원본 데이터는 용량이 매우 크므로, 학습에는 로컬에 확보한 Validation 원천데이터를 사용했다. 폴더명과 중증도 라벨을 기반으로 CSV를 생성하고, train/validation/test split을 구성했다.

## 4. 지원 질환과 데이터 구성

본 프로젝트는 정상 클래스를 포함해 총 11개 클래스를 다중분류한다.

| 구분 | 클래스 |
|---|---|
| 정상 | 정상 |
| 질환 | 결막염, 안검염, 유루증, 핵경화, 백내장, 궤양성 각막질환, 비궤양성 각막질환, 색소침착성 각막염, 안검내반증, 안검종양 |

다중분류 학습용 CSV인 `eye_multiclass_dataset.csv`는 총 25,113개 샘플로 구성했다.

| split | 샘플 수 |
|---|---:|
| train | 17,574 |
| validation | 3,768 |
| test | 3,771 |

또한 질환별 성능 확인을 위해 10개 질환 각각에 대한 이진 분류 CSV와 모델도 별도로 생성했다.

## 5. 전처리

이미지는 EfficientNet-B0 입력 규격에 맞춰 224 x 224 크기로 변환했다. 학습 단계에서는 좌우 반전, 회전, 밝기 및 대비 조정 등 기본적인 이미지 증강을 적용했다. 추론 단계에서는 ImageNet 평균과 표준편차를 사용해 정규화했다.

웹 서비스에서는 모델 추론 전에 다음 품질 항목도 확인한다.

- 해상도: 최소 224px 이상인지 확인
- 눈 영역: 동공/각막으로 보이는 어두운 영역과 주변 명암 차이 확인
- 노출: 과도한 밝기 또는 어두움 여부 확인
- 선명도: 흐림 정도 확인

## 6. 모델 구조

AIHub에서 제공한 `AI 모델 소스코드`는 TensorFlow/Keras 기반 EfficientNetB0/ResNet 전이학습 예제였다. 본 프로젝트는 같은 설계 원리를 PyTorch로 재구현했다.

| 항목 | 구현 |
|---|---|
| Backbone | EfficientNet-B0 |
| Pretraining | ImageNet pretrained weights |
| 입력 크기 | 224 x 224 RGB |
| Loss | CrossEntropyLoss |
| Optimizer | AdamW |
| 출력 | softmax 기반 클래스 확률 |
| 저장 | 검증 성능 기준 best `.pt` 모델 |

초기에는 질환별 이진 분류 모델의 positive 확률을 비교해 상위 질환을 표시했다. 그러나 실제 테스트 중 한 이미지에서 여러 이진 모델이 동시에 높은 확률을 내며 질환 간 순위가 흔들리는 문제가 발견되었다. 이를 개선하기 위해 정상 + 10개 질환을 한 번에 비교하는 다중분류 모델을 추가했고, 현재 웹 서비스의 상위 3개 질환은 다중분류 모델을 기준으로 표시한다.

## 7. 구현 기능

Django 기반 웹 애플리케이션으로 다음 기능을 구현했다.

- 이미지 업로드
- 업로드 직후 사각형 영역 안에 이미지 미리보기
- 예측 실행 버튼 기반 추론
- 이미지 품질 점검 결과 표시
- 정상/질환 의심 상태 표시
- 상위 3개 질환 후보만 표시
- 최고 확률 질환에 대한 설명 표시
- 최고 확률 질환의 예방법 및 관리 방법 표시
- Postman 또는 curl 테스트용 API 제공

API 엔드포인트는 다음과 같다.

```text
POST /api/predict/
```

요청 형식은 `multipart/form-data`이며, `image` 필드에 JPG, JPEG, PNG 파일을 전송한다.

## 8. 성능 평가

질환별 이진 모델은 테스트셋 기준 Accuracy, Precision, Recall, F1 Score를 계산했다. 모든 지원 질환에서 계획서 목표였던 F1 Score 0.8 이상을 달성했다.

| 질환 | Accuracy | Precision | Recall | F1 Score |
|---|---:|---:|---:|---:|
| 결막염 | 0.9833 | 0.9793 | 0.9875 | 0.9834 |
| 안검염 | 0.9966 | 1.0000 | 0.9932 | 0.9966 |
| 유루증 | 0.9944 | 0.9890 | 1.0000 | 0.9945 |
| 핵경화 | 0.9972 | 0.9945 | 1.0000 | 0.9972 |
| 백내장 | 1.0000 | 1.0000 | 1.0000 | 1.0000 |
| 궤양성 각막질환 | 1.0000 | 1.0000 | 1.0000 | 1.0000 |
| 비궤양성 각막질환 | 1.0000 | 1.0000 | 1.0000 | 1.0000 |
| 색소침착성 각막염 | 1.0000 | 1.0000 | 1.0000 | 1.0000 |
| 안검내반증 | 0.9890 | 0.9787 | 1.0000 | 0.9890 |
| 안검종양 | 0.9887 | 1.0000 | 0.9770 | 0.9884 |

다만 질환별 이진 모델의 높은 성능은 같은 질환의 양성/음성 폴더를 구분하는 기준에서 계산된 것이다. 여러 질환 중 하나를 고르는 실제 웹 추론에서는 질환 간 상대 비교가 더 중요하므로, 최종 서비스에는 다중분류 모델을 적용했다.

## 9. 시행착오와 개선

프로젝트 진행 중 가장 중요한 시행착오는 질환별 이진 모델만으로는 상위 질환 순위를 안정적으로 고르기 어렵다는 점이었다. 사용자가 궤양성 각막질환 폴더의 이미지를 업로드했을 때 기존 웹 서비스가 백내장을 1순위로 표시하는 문제가 있었다.

원인을 확인한 결과, 각 이진 모델은 자기 질환 여부만 판단하므로 다른 질환 이미지에도 높은 positive 확률을 줄 수 있었다. 따라서 최종 질환 순위 산정 방식을 정상 + 10개 질환 다중분류 모델로 교체했다. 이후 동일 예시 이미지에서 궤양성 각막질환이 1순위로 표시되도록 개선되었다.

또한 업로드 이미지가 `crop_`으로 시작하는 AIHub 파생 이미지인 경우 원본 파일명을 찾아 학습 데이터 분포에 가까운 원본 이미지를 추론에 사용하도록 보정했다.

## 10. 한계

본 프로젝트는 학습 데이터와 유사한 촬영 조건에서 더 안정적으로 동작한다. 일반 스마트폰 사진은 조명, 거리, 초점, 털 가림, 눈물, 반사광 등 변수가 많아 확률이 낮게 나오거나 유사 질환으로 혼동될 수 있다.

또한 실제 수의학 진단에는 병력, 양안 비교, 통증 반응, 세극등 검사, 안압 검사 등 추가 정보가 필요하다. 따라서 본 시스템은 병원 방문 판단을 돕는 참고 도구로 사용해야 한다.

## 11. 향후 개선 방향

- 눈 영역 자동 검출 또는 segmentation 모델 추가
- 반려견/반려묘, 촬영 장비, 눈 위치를 반영한 메타데이터 결합
- 질환 간 혼동 행렬 분석 및 클래스 불균형 보정

- Grad-CAM 등 설명 가능한 AI 시각화 추가
- 모바일 환경 업로드 UX 개선

## 12. 결론

본 프로젝트는 계획서에서 제시한 이미지 업로드, 전처리, CNN 기반 전이학습, 질환 확률 출력, 설명 제공 흐름을 실제 작동하는 웹 서비스로 완성했다. AIHub 제공 모델 코드를 참고하되 PyTorch와 Django 기반으로 재구현했고, 실제 테스트 중 발견한 질환 순위 문제를 다중분류 모델로 개선했다.

따라서 본 결과물은 딥러닝 모델 학습뿐 아니라 데이터 처리, 모델 평가, 웹 서비스 구현, 사용자 중심 결과 제공까지 포함하는 종합적인 딥러닝 실습 프로젝트로 볼 수 있다.
